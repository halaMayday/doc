# 代码加锁你加对了吗？

先说结论：

是用锁的时候，你需要注意的几件事：

①你弄清楚了我们需要保护的是什么逻辑，多线程执行的情况了吗 ？

②你弄清楚了锁和需要保护的对象是否已一个层次的吗 ？

③你有没有考虑锁的粒度和场景的问题 ？

④你有没有考虑多把锁的死锁问题？

### 1 明确需要保护的逻辑  

我们在使用锁解决问题之前，一定要先弄清楚，我们需要保护的是什么逻辑，多线程执行的情况又是什么样的？

##### 1.1 场景描述  
  在一个类里面有int类型的字段a和b，有一个add方法循环1w次对a和b进行++操作，有另外一个compare方法，同样循环1万次判断a和b的大小关系，判断a和b是否始终相等。


```
public class Demo1 {

    public static void main(String[] args) {
        AddAndCompare addAndCompare = new AddAndCompare();
        new Thread(() -> addAndCompare.add()).start();
        new Thread(() -> addAndCompare.compare()).start();
    }

    public static class AddAndCompare {
        volatile int a = 1;
        volatile int b = 1;

        public void add(){

            System.out.println("start add===>>>");
            for (int i = 0; i < 10000; i++) {
                a++;
                b++;
            }
            System.out.println("<<<===complete add");
        }

        public void compare(){
            System.out.println("start compare===>>>");
            for (int i = 0; i < 10000; i++) {
                if (a < b) {
                    System.out.println("a<b: a的值为："+a+"  b的值为："+b);
                }
            }
        }
    }
}
```

##### 1.2 结果：

```
start add===>>>
start compare===>>>
a<b: a的值为：5724  b的值为：5881
a<b: a的值为：6842  b的值为：6884
a<b: a的值为：7475  b的值为：7660
a<b: a的值为：8238  b的值为：8264
a<b: a的值为：9452  b的值为：9515
<<<===complete add
```

##### 1.3 分析和解决问题

很显然这个是不符合线程安全的代码写法。如何解决呢？加锁。

首先要分析，这两个线程是在交错执行add和compare方法中的业务逻辑的，这些业务逻辑不是原子性的。a++和b++的操作可以穿插在compare方法的比较代码中，并且 a < b 这个比较操作在字节码的层面 是分为 加载a ,加载b ，和比较三步的。

正确处理办法应该是给 add和compare都加上锁。单独只加一个是没办法解决问题的。

```
public synchronized void add(){}
public synchronized void compare(){}
```

**上面的demo1告诉我们，在使用锁解决问题的之前，要先理解清楚，我们需要保护的逻辑是什么，多线程执行的情况又是怎么样的？**

### 2 加锁前弄清楚锁和被保护的对象是否为同一层次

##### 2.1 场景描述

​	在一个类中，我定义了一个静态的int字段 counter，还定义两个非静态的方法 reset和add。add方法我用了锁去保护，那么在多线程循环调用的情况下，是否会出现线程安全问题呢？

```
public class Demo2 {

    public static void main(String[] args) {
        Test.reset();
        IntStream.rangeClosed(1,1000000).parallel().forEach(i -> new Test().add());
        System.out.println(Test.getCounter());
    }

    public static class Test{

        @Getter
        private static int counter = 0;
        public static int reset(){
            counter = 0;
            return counter;
        }
        public synchronized void add(){
            counter++;
        }
    }
}
//结果输出为：197856  与我们期待的1000000不相同

```

##### 2.2 分析和解决问题

**原因是：静态字段属于类，只有类级别的锁才能保护;而非静态字段属于类实例，实例级别的锁就可以保护。**

在非静态的方法add上加锁，只能确保多个线程无法执行同一个实例的add方法。但是不能保证不会去执行不同实例的add方法。而且静态的变量counter在多个实例中共享，所以必然会出现线程安全问题。

解决方法：

①将add方法改为静态，这个时候锁的级别为类级别的。

​	但是为了解决线程的安全问题，就去改变代码结构，这个是不太可取的，不值得推荐。

②可以在类中定义一个Object类型的静态字段，通过锁住这个字段，在每次操作counter之前对它加锁。

```
public static class Test{
        @Getter
        private static int counter = 0;
        private static Object locker = new Object();
        public static int reset(){
            counter = 0;
            return counter;
        }
        public  void add(){
            synchronized (locker) {
                counter++;
            }
        }
    }
```
